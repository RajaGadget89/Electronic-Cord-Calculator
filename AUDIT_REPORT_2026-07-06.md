# รายงานตรวจสอบความถูกต้องของระบบคำนวณ (Audit Report)

**วันที่:** 6 กรกฎาคม 2569 · **ขอบเขต:** engine (app-core + app/src/engine) + UI + รายงาน PDF
**แหล่งอ้างอิงหลัก (source of truth):** คู่มือการติดตั้งระบบไฟฟ้าอย่างมืออาชีพ (EIT Standard 2564) โดย ลือชัย ทองนิล — ไฟล์ PDF ที่แนบ (เลขหน้าอ้างอิงตามเลขหน้าที่พิมพ์ในเล่ม)
**วิธีตรวจ:** ถอดตารางจากหนังสือแยกอิสระจากโค้ด (ยืนยันด้วยภาพหน้าจริง) → เทียบทีละค่า → รันเทสต์เดิม 23/23 → สร้าง harness ตรวจอิสระ 551,198 รายการ ครอบคลุม sweep 86,751 เคส (ทุกชนิดสาย × กลุ่มติดตั้ง × เฟส × กระแส × อุณหภูมิ × จำนวนวงจร × ความยาว) + ตัวอย่างโจทย์จริงจากหนังสือ + edge cases

## สรุปผู้บริหาร

ส่วนที่เป็น "หัวใจ" ของแอพ (ตารางกระแส 5-20/5-21/5-23, ตัวคูณ Ca, ตารางแรงดันตก, สูตรคำนวณกระแสและ %VD, ขั้นตอน It ≥ In/(Ca×Cg)) **ถูกต้องตรงกับหนังสือทุกค่า** และผลลัพธ์ตรงกับตัวอย่างโจทย์ในหนังสือ (ตย. 2.1, 2.2, 9.1) ทุกข้อ

แต่พบบัค/ความต่างจากมาตรฐาน **10 ประเด็น** โดย **5 ประเด็นเป็นระดับอันตราย (แนะนำสายเล็กกว่าที่มาตรฐานกำหนดได้ในบางเงื่อนไข)** — ยังไม่ได้แก้ไขโค้ดใด ๆ ตามที่ตกลง รอคุณยืนยันก่อน

## 🔴 บัคระดับ P0 — อาจแนะนำสายเล็กกว่าที่มาตรฐานกำหนด (อันตราย)

### BUG-1: ตีความคอลัมน์ตาราง 5-22 ผิดความหมาย (บนลูกถ้วย กลุ่ม 4)
- **ไฟล์:** `wireData.ts` บรรทัด 125–128 (`T5_22_single` คอมเมนต์ "higher col = 2cc(1φ), lower col = 3cc(3φ)")
- **ความจริงในหนังสือ (หน้า 220, ดูรูปการติดตั้งหัวตาราง):** สองคอลัมน์คือ **"วางแนวตั้ง" (ค่าต่ำ 30/39/56/78/113...) กับ "วางแนวนอนมีระยะห่าง" (ค่าสูง 37/48/67/92/127...)** — ไม่เกี่ยวกับจำนวนเฟส และตัวอย่างในหนังสือ (หน้า 142, 148) เลือกคอลัมน์ค่าต่ำแม้เป็นงาน 1 เฟส
- **ผลกระทบ:** งาน 1 เฟสบนลูกถ้วยที่วางสายแนวตั้ง แอพให้ค่าสูงเกินจริง เช่น 25 ตร.มม. แอพใช้ 127 A แต่ควรเป็น 113 A (+12%) → เลือกสายเล็กไป 1 ขนาดได้
- **วิธีแก้ที่เสนอ:** ใช้คอลัมน์ค่าต่ำ (แนวตั้ง) เป็น default ทั้ง 1P/3P เพื่อความปลอดภัย หรือเพิ่มตัวเลือก "แนวตั้ง/แนวนอน" ให้ผู้ใช้เลือก

### BUG-2: VCT ใช้ตารางกระแสผิดตาราง
- **ไฟล์:** `wireData.ts` (`DEFAULT_INSTALL_GROUP.VCT = 4` + `lookupAmpacity` + `coreOf`)
- **ความจริงในหนังสือ:** ตาราง 5-22 ระบุใช้กับ **"60227 IEC 01, NYY" เท่านั้น** (หน้า 220) — สาย VCT (สายอ่อน) เดินในอากาศต้องใช้ **ตาราง 5-26** (หน้า 223) ซึ่งค่าต่ำกว่ามาก และแยกคอลัมน์ 1–2 แกน / 3–5 แกน
- **ผลกระทบ (default ของแอพเอง!):** เช่น VCT 10 ตร.มม. 1 เฟส แอพให้ 67 A แต่ตาราง 5-26 ให้ 51 A (2 แกน) หรือ 47 A (3–5 แกน) → **สูงเกินจริง 31–43%**
- **เสริม:** ใน 5-20 (ร้อยท่อ) แอพจัด VCT เป็น "แกนเดียว" แต่ VCT ที่ใช้งานทั่วไปเป็นหลายแกน → ค่าสูงเกินอีก ~7–11% (เช่น 10mm²: 50 vs 45 A) และเทสต์เดิม `calc.test.ts` บรรทัด 112–120 ล็อกพฤติกรรมผิดนี้ไว้ (คาดหวัง 67A)
- **วิธีแก้ที่เสนอ:** เพิ่มตาราง 5-26 สำหรับ VCT ในอากาศ, จัด VCT เป็นหลายแกนใน 5-20 (หรือถามจำนวนแกน), แก้เทสต์ให้อิงหนังสือ

### BUG-3: ไม่มีกฎ 125% สำหรับโหลดมอเตอร์
- **ไฟล์:** `calc.ts` (`selectBreakerA`, `requiredIt`)
- **ความจริงในหนังสือ (ข้อ 6.1.1 หน้า 162):** "สายไฟฟ้าของมอเตอร์ต้องมีขนาดกระแส **ไม่ต่ำกว่า 1.25 เท่า** ของกระแสโหลดเต็มที่" และหน้า 142 แนะนำเมนสวิตช์ ≥ 1.25×กระแสโหลด
- **ผลกระทบ:** แอพมี preset ปั๊มน้ำ/มอเตอร์, แอร์, พัดลม, ตู้เย็น (โหลดมอเตอร์ทั้งหมด) แต่คำนวณด้วยกระแสตรง ๆ เช่น มอเตอร์ FLC 40 A: มาตรฐานต้องการสายรับ ≥ 50 A → 16 ตร.มม. (60 A) แต่แอพให้ **10 ตร.มม. (45 A)** — เล็กไป 1 ขนาด ขณะมอเตอร์เดินต่อเนื่อง/สตาร์ทหนักจะร้อนเกิน
- **วิธีแก้ที่เสนอ:** โหลดชนิดมอเตอร์ (pump/ac/fan/fridge) ให้คูณ 1.25 เข้ากับกระแสที่ใช้กำหนดสายและเบรกเกอร์ (หรืออย่างน้อยแสดงคำเตือน + ปุ่มเปิดใช้กฎมอเตอร์)

### BUG-4: Cg ฝังดินโดยตรงหลายวงจร ใช้ตารางผิด → derate น้อยเกินไป
- **ไฟล์:** `wireData.ts` (`groupingFactor` ใช้ตาราง 5-8 กับทุกกลุ่ม)
- **ความจริงในหนังสือ (หมายเหตุ 2 ตาราง 5-23 หน้า 221):** ฝังดิน >1 วงจร ให้ใช้ **ตาราง 5-45 (ฝังดินตรง) / 5-46 (ร้อยท่อฝังดิน)** หน้า 250
- **ผลกระทบ:** กลุ่ม 6 วางชิดกัน: หนังสือ 0.75/0.65/0.60/0.55/0.50 (2–6 วงจร) แต่แอพใช้ 0.80/0.70/0.65/0.60/0.57 → **derate น้อยไปทุกช่อง → สายเล็กเกิน** (กลุ่ม 5 แอพเผอิญ conservative กว่า 5-46 → ปลอดภัยแต่เปลืองเงิน)
- **วิธีแก้ที่เสนอ:** แยกตาราง Cg ตามกลุ่ม: 1/2→5-8 คอลัมน์ 1, 3→5-8 คอลัมน์ 2, 5→5-46, 6→5-45 (ใช้คอลัมน์ "วางชิดกัน" เป็น default ปลอดภัยสุด)

### BUG-5: อุณหภูมิเกิน 60°C — มาตรฐานห้ามใช้สาย PVC แต่แอพคำนวณต่อ
- **ไฟล์:** `wireData.ts` (`ambientFactor` fallback ค่าแถวสุดท้าย), `calc.ts` validation (รับ −49 ถึง 199°C)
- **ความจริงในหนังสือ (ตาราง 5-43/5-44):** เกิน 60°C ช่อง PVC 70°C เป็น **"-"** (ไม่มีตัวคูณ = ห้ามใช้งาน)
- **ผลกระทบ:** กรอก 65°C (หรือ 199°C!) แอพยังให้ผล **PASS** ด้วย Ca = 0.57
- **วิธีแก้ที่เสนอ:** ambientTempC > 60 → FAIL พร้อมข้อความ "เกินขอบเขตสาย PVC 70°C ปรึกษาวิศวกร" (ต่ำกว่า 11°C ใช้ 1.34 ต่อได้ = ฝั่งปลอดภัย)

## 🟠 บัคระดับ P1 — ไม่อันตราย แต่ผิดจากมาตรฐาน (ผู้ใช้จ่ายแพงเกินจำเป็น/ความน่าเชื่อถือ)

### BUG-6: ตารางสายดินไม่ตรงตาราง 4.2 ของหนังสือ (6 จาก 12 พิกัด)
- **ไฟล์:** `wireData.ts` (`EQUIP_GROUND` — และคอมเมนต์อ้าง "ตารางที่ 6-11" ซึ่งไม่มีในหนังสือ; ตารางจริงคือ **4.2 หน้า 115**)
- **ค่าที่ต่าง (เบรกเกอร์ → แอพ / หนังสือ):** 16A → 4 / **2.5** · 20A → 4 / **2.5** · 25A → 6 / **4** · 32A → 6 / **4** · 40A → 6 / **4** · 63A → 10 / **6** (พิกัด 6,10,50,80,100,125A ตรงกัน; แถว "150 ตร.มม." ในแอพไม่มีอยู่จริงในตาราง 4.2)
- ทิศทางผิดแบบ "ใหญ่เกิน" = ไม่อันตราย แต่ไม่ตรงมาตรฐานและเปลืองต้นทุนผู้ใช้ + เทสต์เดิมบรรทัด 181–190 ล็อกค่าผิดไว้ (คาดหวัง 40A→6)
- **วิธีแก้ที่เสนอ:** เปลี่ยน `EQUIP_GROUND` เป็นตาราง 4.2: `[{20:2.5},{40:4},{70:6},{100:10},{200:16},{400:25},{500:35},{800:50},{1000:70},{1250:95},{2000:120},{2500:185},{4000:240},{6000:400}]` และแก้คอมเมนต์อ้างอิง

### BUG-7: Cg กลุ่ม 3 (เกาะผนัง) ควรใช้คอลัมน์ 2 ของตาราง 5-8
- **ความจริงในหนังสือ (ตาราง 5-8 หน้า 217 + หมายเหตุ 3 ตาราง 5-21):** เดินเกาะผนัง/บนผิว ใช้คอลัมน์ 2 (0.85/0.79/0.75/0.73/0.72...) แต่แอพใช้คอลัมน์ 1 (0.80/0.70/0.65/0.60/0.57...) กับทุกกลุ่ม
- **ผลกระทบ:** VAF หลายวงจรถูกบังคับใช้สายใหญ่เกินจำเป็น (เช่น 6 วงจร: 0.57 แทน 0.72 = ต้องเผื่อเกิน ~26%)

## 🟡 บัคระดับ P2 — ข้อความ/การตรวจสอบข้อมูล

### BUG-8: VAF + 3 เฟส ให้ข้อความชวนเข้าใจผิด
เลือก VAF แล้วสลับเป็น 3 เฟสได้ใน UI → engine ตอบ "ค่าพิกัดกระแส…ยังไม่ยืนยัน (รอการตรวจสอบ)" ซึ่งไม่จริง — ความจริงคือ **VAF เป็นสายแบน 1 เฟส ใช้กับ 3 เฟสไม่ได้** ควรบอกตรง ๆ และ/หรือ UI ล็อกไม่ให้เลือกคู่นี้

### BUG-9: ช่องรับข้อมูลหลวมเกิน
จำนวนกลุ่มวงจรรับทศนิยม (เช่น 2.5), อุณหภูมิรับช่วง −49…199°C (เกี่ยวโยง BUG-5), และกรณี mv เป็น null แรงดันตกถูกบันทึกเป็น 0% แทน "ไม่ทราบ" (`calc.ts` บรรทัด 139 `round(vd ?? 0, 2)`)

### BUG-10 (เอกสาร): README ระบุ "ผ่าน 21/21" แต่ปัจจุบันมี 23 เทสต์ และเทสต์ 2 ข้อล็อกพฤติกรรมที่ผิดมาตรฐานไว้ (ดู BUG-2, BUG-6)

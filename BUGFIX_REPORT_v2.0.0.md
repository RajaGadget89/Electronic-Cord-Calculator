## บัคที่พบและการแก้ไข

### 🔴 P0 — เดิมอาจแนะนำสายเล็กกว่ามาตรฐาน (กระทบความปลอดภัย)

**BUG-1 · ตีความคอลัมน์ตาราง 5-22 ผิด (บนลูกถ้วย กลุ่ม 4)**
โค้ดเดิม map สองคอลัมน์เป็น 1φ/3φ แต่ความจริงคือ "วางแนวตั้ง (ค่าต่ำ)" กับ "วางแนวนอนมีระยะห่าง (ค่าสูง)" (รูปหัวตาราง หน้า 220) ทำให้งาน 1 เฟสได้ค่าสูงเกิน ~12% เช่น 25 mm² ใช้ 127 A แทน 113 A
**แก้:** ใช้คอลัมน์แนวตั้ง (ค่าต่ำ = ปลอดภัย) กับทุกเฟส (`T5_22_VERTICAL`) เก็บคอลัมน์แนวนอน export ไว้เผื่อเพิ่มตัวเลือกภายหลัง — `wireData.ts`

**BUG-2 · VCT ใช้ตารางกระแสผิด**
(a) กลุ่ม 4 เดิมอ่านตาราง 5-22 ซึ่งหัวตารางระบุใช้กับ "60227 IEC 01, NYY" เท่านั้น → VCT 10 mm² ได้ 67 A ทั้งที่ตารางสายอ่อน 5-26 (หน้า 223) ให้ 51 A (2 แกน) / 47 A (3–5 แกน) — สูงเกินจริงถึง 43%
(b) ใน 5-20 เดิมจัด VCT เป็น "แกนเดี่ยว" ทั้งที่ VCT ใช้งานทั่วไปเป็นหลายแกน — สูงเกิน ~7–11%
**แก้:** เพิ่มตาราง `T5_26_VCT` (1φ→คอลัมน์ 1–2 แกน, 3φ→คอลัมน์ 3–5 แกน), `coreOf(VCT)` → `multi`, label วิธีติดตั้งของ VCT ใน UI เปลี่ยนเป็น "เดินในอากาศ (ต่อเครื่องใช้ไฟฟ้า)" — `wireData.ts`, `JobForm.tsx`

**BUG-3 · ไม่มีกฎ 125% สำหรับโหลดมอเตอร์ (วสท. ข้อ 6.1.1 หน้า 162)**
เดิมมอเตอร์ FLC 40 A ได้สาย 10 mm² (45 A) — มาตรฐานต้องการสายรับ ≥ 50 A (16 mm²)
**แก้:** เพิ่ม `designCurrentA()` = โหลดปกติ 100% + โหลดมอเตอร์ ×1.25 ใช้กำหนดทั้งเบรกเกอร์และสาย, เพิ่มฟิลด์ `isMotor` ใน `LoadType`/`LoadItem` (preset ปั๊ม/แอร์/พัดลม/ตู้เย็น = มอเตอร์; งานเก่าที่บันทึกไว้ fallback จาก id), UI มี checkbox "โหลดมอเตอร์" ตอนเพิ่มชนิดโหลดเอง, แสดง "กระแสออกแบบ" ใน ResultCard/รายงาน/PDF พร้อมคำอธิบายใน warnings
หมายเหตุการออกแบบ: มาตรฐานกำหนด 1.25×ตัวใหญ่สุด + ตัวอื่น 100% — แอปคูณ 1.25 กับมอเตอร์ทุกตัว ให้ผล ≥ มาตรฐานเสมอ (ฝั่งปลอดภัย, ระบุในคอมเมนต์โค้ด)

**BUG-4 · Cg ฝังดินหลายวงจรใช้ตารางผิด**
เดิมใช้ตาราง 5-8 กับทุกกลุ่ม แต่หมายเหตุ 2 ของตาราง 5-23 (หน้า 221) ให้ใช้ตาราง 5-45/5-46 (หน้า 250) — กลุ่ม 6 เดิม derate น้อยไป (เช่น 2 วงจร: 0.80 แทน 0.75) → สายเล็กเกิน
**แก้:** `groupingFactorEx(n, group)` แยกตาราง: กลุ่ม 1/2/4 → 5-8 คอลัมน์ 1, กลุ่ม 3 → 5-8 คอลัมน์ 2, กลุ่ม 5 → 5-46, กลุ่ม 6 → 5-45 (คอลัมน์ "วางชิดกัน" = ค่าปลอดภัยสุด) · เกินขอบเขตตาราง (ฝังดิน >6 วงจร) ใช้ค่าต่ำสุด + เตือน + สถานะไม่เกิน WARN — `wireData.ts`, `calc.ts`

**BUG-5 · อุณหภูมิเกิน 60°C ยังคำนวณต่อ**
ตาราง 5-43/5-44 ช่วง >60°C เป็น "-" (ห้ามใช้สาย PVC) แต่เดิม validation รับถึง 199°C แล้วใช้ Ca=0.57 → ได้ PASS
**แก้:** `MAX_AMBIENT_C = 60`, validate เกินแล้ว FAIL พร้อมข้อความแนะนำ, UI validate + ปรับ help text (ต่ำกว่า 11°C ยังใช้ 1.34 = ฝั่งปลอดภัย) — `calc.ts`, `JobForm.tsx`

### 🟠 P1 — ไม่อันตราย แต่ผิดจากมาตรฐาน (ผู้ใช้จ่ายแพงเกิน)

**BUG-6 · ตารางสายดินไม่ตรงตาราง 4.2 (หน้า 115) + อ้างเลขตารางผิด**
เดิม 6/12 พิกัดให้สายดินใหญ่เกิน (16A→4 ควร 2.5 · 20A→4 ควร 2.5 · 25/32/40A→6 ควร 4 · 63A→10 ควร 6) และคอมเมนต์อ้าง "ตาราง 6-11" ซึ่งไม่มีจริง (ที่ถูกคือ วสท. ตารางที่ 4-2)
**แก้:** เปลี่ยน `EQUIP_GROUND` เป็นค่าตาราง 4.2 ครบ 14 แถว (20→2.5 … 6000→400), แก้การอ้างอิงทุกจุด รวมถึงในคู่มือ PDF หน้า 2

**BUG-7 · Cg กลุ่ม 3 (เกาะผนัง) ใช้คอลัมน์ผิด**
เดิมใช้คอลัมน์ 1 (0.80/0.70/…) ที่เข้มกว่าจำเป็น — คอลัมน์ 2 ของตาราง 5-8 (0.85/0.79/0.75/0.73/0.72…) คือคอลัมน์สำหรับเดินเกาะผนัง → แก้แล้วใน `groupingFactorEx` (รวมกับ BUG-4)

### 🟡 P2 — ข้อความ / validation / เอกสาร

**BUG-8 · VAF + 3 เฟส ให้ข้อความชวนเข้าใจผิด** ("รอการตรวจสอบ") → validate เป็น error ชัดเจน "VAF เป็นสายแบนสำหรับวงจร 1 เฟสเท่านั้น…" + UI ปิดตัวเลือก 3 เฟสเมื่อเลือก VAF และสลับกลับ 1 เฟสอัตโนมัติ

**BUG-9 · Validation หลวม** → จำนวนกลุ่มวงจรต้องเป็นจำนวนเต็ม, ค่าโหลดต้อง finite (กัน Infinity/NaN), แรงดันตกที่ไม่ทราบค่าเก็บเป็น `null` (เดิมกลายเป็น 0%) และสถานะจะไม่เป็น PASS หากไม่ทราบ %VD

**BUG-10 · เอกสารเลขเทสต์ไม่ตรง** → README อัพเดตเป็น 39/39 + วิธีรัน `audit.ts`

## สิ่งที่ยืนยันว่า "ถูกต้องอยู่แล้ว" (ไม่แตะ)

ตาราง 5-20 (80 ค่า), 5-21, 5-23 (30 ค่า), Ca 5-43/5-44 ทุกช่วง, ตารางแรงดันตก 9.1/9.2 (40 ค่า — เลือกคอลัมน์ conservative), สูตร W→A (1φ/3φ), สูตร %VD ฐาน 230/400, หลัก It ≥ In/(Ca·Cg), กติกาสายดินไม่ใหญ่กว่าสายเฟส, เกณฑ์ VD 3% (เข้มกว่าเกณฑ์รวม 5% ของมาตรฐาน = ฝั่งปลอดภัย) และผลตรงกับตัวอย่างโจทย์ในหนังสือ ตย. 2.1, 2.2, 9.1 ทุกข้อ
